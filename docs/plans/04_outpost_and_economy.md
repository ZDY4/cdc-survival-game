# 据点与经济回路计划

## 目标

让 `survivor_outpost_01` 不只是出生点，而是真正承担以下职责的中心节点：

- 接任务
- 交易
- 整理背包
- 存取物资
- 制作与准备
- 从失败中恢复
- 为下一次外出提供动机

## 当前现状

- 据点地图内容已经明显比普通地图丰富。
- 有容器、拾取物、trigger、室内/外切换、商人角色和聚落定义。
- UI 已经支持交易、容器和制作面板。
- `shops` 目前仅有 1 个商店，适合作为第一版经济中心。

## 关键问题

- 据点功能存在，但尚未被组织成玩家主循环。
- 交易、容器、制作和任务奖励之间还没有被统一设计。
- 玩家从外场回来后可能只是在 UI 上“能点开功能”，却没有被驱动去使用这些功能。

## 本阶段范围

- 只围绕第一个据点做主循环。
- 只先打通一个经济回路：
  - 外出带回资源
  - 回据点交易/存放/制作
  - 获得更适合下一次外出的准备

## 不做什么

- 不做完整基地建设系统。
- 不做复杂居民满意度或日夜经营。
- 不做多商店、多货币体系。

## 推荐的据点循环

1. 玩家回到据点。
2. 交付任务。
3. 整理背包或把物资存入容器。
4. 将一部分资源交易成即时需要物资。
5. 根据已解锁配方做 1 次基础制作。
6. 再次出发。

## 首批重点设施

### 商人

职责：

- 提供基本补给回补。
- 作为任务发放或信息节点之一。

要求：

- 商品结构要直接服务前 20 分钟。
- 能买到的东西不能直接跳过外场价值。

### 医疗/补给容器

职责：

- 让玩家在据点内感知“安全区资源组织”。
- 强化容器系统不是只有外场才会使用。

### 工坊或制作入口

职责：

- 让材料、配方和制作第一次产生真实价值。

## 经济设计约束

- 前 20 分钟只需要一个简单闭环，不需要完整经济模拟。
- 玩家应该始终觉得“搜刮来的东西有去处”。
- 交易价格和制作收益不要让任一系统完全压过另一系统。

## 失败恢复挂接

据点应承担玩家失败后的恢复作用，至少需要定义：

- 失败后回到哪里。
- 丢失什么。
- 保留什么。
- 如何重新出发。

如果没有据点恢复逻辑，外场失败只能是 runtime 层的结果，不是游戏层的流程。

## 任务拆分

### A. 明确据点中的每个核心功能点

- 谁负责交易。
- 哪些容器是公共补给。
- 玩家在哪触发制作入口。
- 玩家失败后回到哪里。

### B. 收口第一版经济节奏

- 定义首批核心资源。
- 定义首批关键消耗品。
- 定义首批交易价值参考。

### C. 让任务奖励与据点消费衔接

- 奖励不能只是数字。
- 奖励最好直接推动一次交易、制作或地点解锁。

## 验收标准

- 玩家第一次外场返回后，会自然使用至少两项据点功能。
- 交易、容器或制作中至少有一项成为玩家真实决策。
- 玩家能从据点获得下一次外出的准备，而不是只读一段状态文本。

## 风险

- 如果据点只是 UI 面板集合，玩家不会把它理解为主循环中心。
- 如果交易收益过强，会压掉搜刮和制作。
- 如果制作价值过弱，会让配方和材料系统前期失去意义。
