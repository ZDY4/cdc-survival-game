# 主线后续打磨项

当前范围只覆盖现有首段主线完成之后的下一段内容，不回头重复已经打通的：

- `tutorial_survive`
- `zombie_hunter`
- `find_medicine`

本文件只保留接下来尚未完成的后续打磨项。

## 当前基线

目前已确认可跑的第一段主线是：

1. 老王发放 `tutorial_survive`
2. 老王衔接 `zombie_hunter`
3. 陈医生发放并回收 `find_medicine`

这意味着“据点外试跑 -> 清外围 -> 医院取药 -> 回据点交付”的第一段已经成型。

## 当前剩余问题

### 1. 医院段结束后没有下一步驱动

`find_medicine` 完成后，玩家已经能回到据点，但暂时没有新的明确主线目标。

这会导致：

- 第一段结束后主线断档。
- 玩家知道系统能玩，但不知道接下来该去哪。

### 2. 第二圈层地点还没有被主线正式消费

仓库里已经有：

- `factory`
- `supermarket`
- `school`
- `ruins`
- `subway`
- `forest`

但这些地点还没有被接进当前主线，仍然更像“可去的地图”，不是“必须去的下一站”。

### 3. 据点对医院段完成的后续反应还不够

虽然医院任务已经能交付，但据点还缺少更持续的后续变化，例如：

- 新地点情报
- 新任务交接
- 商店/补给结构的小幅变化
- 明确的“下一段比上一段更远、更危险”的准备信号

### 4. 中段出发前的准备压力还不够明确

第一段现在已经更像 onboarding，但还没逼玩家认真使用：

- 交易
- 制作
- 医疗准备
- 弹药准备

医院之后的下一段应当开始要求玩家在出门前做准备，而不只是继续直推。

## 本轮固定方案

为避免范围扩散，这份后续打磨文档先固定一版下一段主线方案：

- 下一站地点优先选 `factory`
- 任务发放者优先仍由老王承担
- 任务类型继续只用已跑通的 `collect` / `kill` / 手动交付
- 完成 `factory` 段后，再决定是否把 `supermarket` 或 `subway` 接成下一张图

不在这一轮同时铺两条以上主线分支。

## 本轮目标

把现有主线从“医院段结束”推进到“第二圈层第一次长途外出”：

- 玩家完成医院交付后，立刻获得下一步方向
- 玩家知道为什么要去 `factory`
- 玩家会为了这次出发进行一次明确准备
- 玩家回来后能得到下一段驱动，而不是再次断档
