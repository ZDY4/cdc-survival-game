# 负重系统设计方案

## 功能概述
为游戏添加真实的负重机制，影响玩家在大地图中的移动和战斗表现。

---

## 核心机制

### 1. 负重计算

#### 重量来源
| 来源 | 计算方式 |
|------|---------|
| **装备** | 8个装备槽位的总重量 |
| **背包** | 所有物品数量 × 单件重量 |
| **武器** | 主手+副手武器重量 |

#### 负重等级
```
负重率 = 当前重量 / 最大负重 × 100%

轻载:   0% - 50%   (无惩罚)
中载:  50% - 75%   (轻微惩罚)
重载:  75% - 90%   (明显惩罚)
超载:  90% - 100%  (严重惩罚)
完全超载: >100%    (无法移动)
```

---

## 2. 负面效果设计

### 移动惩罚（大地图）
| 负重等级 | 移动时间倍数 | 描述 |
|---------|-------------|------|
| 轻载 | ×1.0 | 正常移动 |
| 中载 | ×1.3 | 步伐稍慢 |
| 重载 | ×1.6 | 气喘吁吁 |
| 超载 | ×2.2 | 步履艰难 |
| 完全超载 | ×5.0 | 几乎无法移动 |

### 战斗惩罚
| 负重等级 | 效果 |
|---------|------|
| 中载 | 闪避率 -10% |
| 重载 | 闪避率 -20%, 先手值 -5 |
| 超载 | 闪避率 -40%, 先手值 -10, 每回合消耗额外体力 |
| 完全超载 | 无法战斗，只能逃跑 |

### 其他惩罚
- **耐力消耗**: 重载以上，移动和战斗额外消耗耐力
- **噪音**: 超载时移动会产生噪音，增加遭遇敌人概率
- **陷阱触发**: 重载以上，触发陷阱概率增加

## 5. 负重计算公式

### 基础公式
```
最大负重 = 基础负重 + 力量加成 + 背包加成 + 装备加成

基础负重: 30 kg (固定)
力量加成: 力量 × 3 kg
背包加成: 背包提供的负重上限
装备加成: 某些护甲/配件提供的负重
```

### 示例
```
角色: 力量 5
装备: 军用背包 (+20kg负重), 战术背心 (+5kg负重)

最大负重 = 30 + (5×3) + 20 + 5 = 70 kg
```

### 背包类型
| 背包类型 | 负重加成 | 描述 |
|---------|---------|------|
| 无背包 | +0 kg | 初始状态 |
| 简易布袋 | +5 kg | 轻量便携 |
| 登山包 | +10 kg | 户外运动 |
| 军用背包 | +20 kg | 专业装备 |
| 战术背包 | +25 kg | 顶级背包 |

**注意**: 背包只有负重加成，没有容量限制。物品数量只受总重量限制。

### 装备负重加成
| 装备 | 类型 | 负重加成 |
|------|------|---------|
| 战术背心 | 身体 | +5 kg |
| 负重腰带 | 腰部 | +3 kg |
| 军用靴 | 脚部 | +2 kg |

## 5. 系统集成

### 存档集成
- 负重数据保存到存档
- 读取时重新计算当前重量

### 事件集成
```gdscript
# 新事件
weight_changed(current, max, ratio)
overload_started()
overload_ended()
```

### 大地图移动集成
```gdscript
# 修改MapModule.travel()
func travel(destination: String) -> float:
    var base_time = get_travel_time(destination)
    var penalty = CarrySystem.get_movement_penalty()
    return base_time * penalty
```

## 6. 平衡性考虑

### 早期游戏
- 初始负重较低 (40-50kg)
- 鼓励玩家做选择：带食物还是带武器
- 增加资源管理的策略性

### 中后期游戏
- 力量提升可增加负重
- 特殊装备/背包可以增加负重上限
- 基地储存解决物品管理问题

### 技巧性玩法
- 轻装快速探索
- 重装慢速但资源丰富
- 临时丢弃物品减重

## 9. 风险与注意事项

⚠️ **风险点**:
1. 需要为大量现有数据添加weight字段
2. 可能影响现有存档兼容性
3. UI改动需要与现有界面协调

✅ **缓解措施**:
1. 没有weight的物品默认0.1kg
2. 存档迁移时自动计算重量
3. UI采用非侵入式设计
