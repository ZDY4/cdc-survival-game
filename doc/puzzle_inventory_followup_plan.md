# 拖拽可放预判高亮方案

## 摘要
- 放弃“仅装备槽用绿/红区分”的收缩方案，改回完整预判方案。
- 拖拽时同时提供两类预判：
  - 装备槽：兼容性预判
  - 背包格子：落点占用预判
- 视觉方案锁定为“混合样式”：
  - 装备槽用边框/底色状态提示
  - 背包格子用半透明覆盖块提示
- 背包格子预览采用“整块统一色”：
  - 可放时，整个物品占用区域统一绿色
  - 不可放时，整个物品占用区域统一红色
  - 不拆分冲突格，不做局部红绿混合

## 关键实现变更
- 在装备槽控件中加入拖拽悬停状态刷新能力。
  - 对 `inventory_item` 和 `equipped_item` 都做兼容性判定。
  - 悬停时进入三态之一：`default`、`valid_drop`、`invalid_drop`。
  - `NOTIFICATION_DRAG_END` 或鼠标离开后恢复默认样式。
- 在背包网格视图中加入“当前拖拽物预览层”。
  - 预览层独立于已放置物品层，不改真实数据。
  - 根据鼠标所在格和拖拽物 footprint，实时计算目标区域。
  - 预览源覆盖三种场景：
    - 背包物品在背包内重排
    - 已装备物品拖回背包
    - 背包物品从背包拖出后又移回背包
- 预览判定沿用现有真实规则，不新增第二套逻辑。
  - 装备槽继续复用现有槽位兼容规则。
  - 背包格子继续复用现有 `_can_place_entry_at()` / `move_item_instance()` 对应的放置规则。
  - 对已装备物品拖回背包时，按“临时从装备槽移除后再判定”的视角做预览，保证预览结果与最终落地一致。
- 新增统一的拖拽预览状态承载。
  - 预览数据至少包含：`type`、`instance_id`、`item_id`、`source_slot`、`footprint`、`origin_kind`。
  - 不修改存档结构；这是纯 UI 运行时状态。
- 装备槽视觉规范固定：
  - 可放：绿色描边，允许叠加轻微绿色底色。
  - 不可放：红色描边，允许叠加轻微红色底色。
  - 非目标状态：回到当前默认主题。
  - 同源槽位不高亮为可放，例如把主手拖回主手时保持默认。
- 背包格子视觉规范固定：
  - 预览区域为与物品 footprint 一致的矩形半透明覆盖块。
  - 可放：绿色半透明覆盖 + 细描边。
  - 不可放：红色半透明覆盖 + 细描边。
  - 只显示一块最终结果，不显示逐格冲突细节。
  - 预览块跟随鼠标所对应的目标格更新，不吸附到非法区域外。
- 不做的内容：
  - 不引入物品旋转。
  - 不做分格冲突细化提示。
  - 不新增背包外的额外浮窗提示。
  - 不修改背包容量、装备规则或实例模型。

## 公开接口与行为约束
- 装备槽按钮脚本需要新增运行时样式接口，形式可为：
  - `set_drop_preview_state(state: String)` 或等价的枚举接口。
- 背包网格视图需要新增运行时预览接口，形式可为：
  - `set_drag_preview(payload, target_cell, is_valid)`
  - `clear_drag_preview()`
- 若当前 Godot 拖拽链路无法从目标控件直接稳定拿到完整拖拽上下文，则允许在背包 UI 层维护一个“当前拖拽物上下文”缓存，并在拖拽开始/结束时同步。
- 实现时必须保证：
  - 预览判定与最终放置结果一致。
  - 拖拽取消、失败、跨控件移动后，所有高亮都能正确清空。
  - 不因为预览刷新触发任何真实的装备/背包状态变更。

## 测试与验收
- 从背包拖头盔时：
  - 头部槽显示可放样式。
  - 其他不兼容槽显示不可放样式。
- 从背包拖饰品时：
  - 两个饰品槽都显示可放样式。
- 从背包拖主手武器时：
  - 主手/副手按当前规则显示可放或不可放。
- 从装备槽拖物品到另一个兼容装备槽时：
  - 目标槽显示可放样式，松手后移动成功。
- 从装备槽拖物品到不兼容装备槽时：
  - 目标槽显示不可放样式，松手后无状态变化。
- 从背包拖一个 `2x3` 物品到网格空位：
  - 背包中显示一个 `2x3` 绿色覆盖块。
- 从背包拖一个 `2x3` 物品到越界、碰撞、未启用区域：
  - 背包中显示一个 `2x3` 红色覆盖块。
- 从装备槽把装备拖回背包：
  - 预览结果与实际 `unequip_to_cell()` 是否成功完全一致。
- 拖拽结束后：
  - 所有装备槽边框恢复默认。
  - 背包预览覆盖块消失。
- 整理背包、切换面板、关闭菜单、拖拽取消后：
  - 不残留任何预览状态。
- 现有拖拽能力不回退：
  - 背包内重排
  - 背包到装备槽
  - 装备槽到背包
  - 装备槽之间互拖

## 假设与默认
- 继续以 `ui/game_menu_overlay.gd` 为唯一有效背包 UI 入口。
- 继续复用当前实例化物品模型，不新增存档字段。
- 继续复用当前装备兼容规则：`main_hand -> off_hand`、`accessory -> accessory_1/2`。
- 视觉实现优先用代码绘制/主题覆盖，不要求新增美术资源。
- 如果同一时刻只能稳定维护一份拖拽上下文，则以当前正在拖拽的单个物品为准，不支持多物品批量拖拽。
